# 术语表

## A

### Agent
智能体，是 OpenCrab 的核心执行单元。每个 Agent 包含 Brain（LLM交互）、推理引擎、工具执行器等组件。

### AgentOrchestrator
多智能体协调器，负责路由消息和任务委派。

## B

### Brain
LLM 交互层，封装了与语言模型的通信逻辑，包括流式输出、工具调用、重试等。

### Blackboard
共享黑板，用于 Agent 间的数据共享和协作。

## F

### FTS5
SQLite 的全文搜索扩展，用于记忆系统的基础检索。

## L

### LLMClient
统一的 LLM 客户端，封装了多种 Provider（Anthropic、OpenAI兼容）。

## M

### MCP (Model Context Protocol)
模型上下文协议，用于扩展 Agent 能力的标准化方式。

### MemoryManager
记忆管理器，负责三层记忆的存储、检索和整合。

## P

### PromptAssembler
提示词组装器，将身份文档、记忆、工具目录等组装成完整的系统提示词。

## R

### Ralph Wiggum 模式
项目的核心设计理念——永不放弃的执行模式。通过状态持久化和断点恢复确保任务完成。

### ReAct (Reason + Act + Observe)
推理引擎使用的模式，显式地进行推理→行动→观察循环。

### Retrieval
多路径记忆检索，结合语义相似度和重排序获取最相关的记忆。

## S

### Skill
技能，一种声明式的能力定义（SKILL.md），可以被 Agent 动态加载和使用。

### Session
会话，用户与 Agent 交互的上下文容器，包含历史消息、状态等。

## T

### ToolExecutor
工具执行器，负责调用具体的工具处理器（ToolHandler）。

### Token Budget
Token 预算，用于控制提示词长度和成本。

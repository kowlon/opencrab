# 运行时数据目录

**位置**: `data/`

**说明**: 项目运行后动态生成的运行时数据存储目录，包含数据库、会话、记忆、日志等。以下是各文件夹/文件的用途及控制它的代码模块。

---

## 目录结构总览

```
data/
├── agent.db              # 主数据库（SQLite）
├── agent.db-shm          # SQLite 共享内存
├── agent.db-wal          # SQLite WAL 模式日志
├── backend.heartbeat     # 后端心跳文件
├── device.json           # 设备 ID
├── llm_endpoints.json    # LLM 端点配置
├── sub_agent_states.json # 子 Agent 状态
├── web_access.json       # Web 访问配置
│
├── agents/               # Agent 配置存储
├── avatars/              # 用户头像
├── delegation_logs/      # 委派日志
├── llm_debug/            # LLM 调试日志
├── memory/               # 记忆系统数据
├── org_templates/        # 组织模板
├── orgs/                 # 组织数据
├── plans/                # 计划文件
├── react_traces/         # ReAct 执行追踪
├── retrospects/          # 回顾日志
├── scheduler/            # 调度器数据
├── selfcheck/            # 自我诊断报告
├── sessions/             # 会话数据
├── sticker/              # 表情包数据
├── traces/               # 执行追踪
└── user/                 # 用户数据
```

# Session Title LLM Generation Design

## Problem

Session titles are generated by truncating the first 30 characters of the user's first message. Two code paths do the same truncation independently:
- Frontend `chat.ts:236` — `addUserMessage()` on first message
- Backend `seecrab.py:175` — SSE `session_title` event emission

The existing LLM-based title generation endpoint `POST /api/sessions/generate-title` (in `sessions.py:205-241`) is fully functional but never called by SeeCrab.

## Design

Keep truncated title as immediate placeholder. After first AI reply completes, fire-and-forget call to `POST /api/sessions/generate-title` with `{ message, reply }`. Replace sidebar title with LLM result.

### Changes

1. **`http-client.ts`** — Add `generateTitle(message, reply)` method calling `/api/sessions/generate-title`
2. **`chat.ts`** — In `case 'done'`, detect first conversation round (`messages.length === 2`), call `generateTitle()`, update session title on success

### No backend changes needed

The existing endpoint uses `brain.think_lightweight()` with prompt: "4-10字中文标题，不加标点引号". Has fallback to 20-char truncation on failure.

### Edge cases

- LLM failure: endpoint returns truncated fallback, frontend catch ignores errors
- Session switch during generation: `.then()` checks activeSessionId still matches
- Non-first-round: guarded by `messages.length === 2`

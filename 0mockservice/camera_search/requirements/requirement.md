# 相机搜索 Mock 服务需求说明（细化版）

## 1. 目标与范围

### 1.1 目标
实现一个用于联调与演示的 Mock 相机搜索服务，提供：
- 基于自然语言的相机搜索能力
- 基于相机 ID 的预处理任务能力（模拟截帧 + 远端处理）
- 基于任务 ID 的预处理状态与进度查询能力
- 基于用户特征描述的图片检索能力（支持文字、图片、图片+文字）

### 1.2 范围
- 仅实现 Mock 能力，不涉及真实摄像头设备接入
- 数据来源为本地配置文件、本地视频文件与本地图片文件
- 接口返回可供上层系统直接消费的结构化结果

## 2. 输入数据规范

### 2.1 相机配置文件
- 服务启动后从指定目录加载相机配置文件
- 每个配置文件至少包含以下字段：
  - `id`：相机唯一标识
  - `location`：地理位置信息（可为文本地址、经纬度描述或二者组合）

### 2.2 视频文件
- 视频目录下包含多个视频文件
- 文件命名格式：`{id}_{地理位置信息}.mp4`
- 其中 `{id}` 应与相机配置中的 `id` 可对应，用于通过相机 ID 定位视频源

## 3. 功能需求

### 3.1 相机搜索（自然语言）
- 输入：用户自然语言描述（示例：`查找园区东门附近的相机`）
- 处理：根据描述在已加载相机集合中进行匹配
- 输出：匹配相机列表，每项至少包含：
  - `id`
  - `location`
  - `name`（相机名称）
  - `image_url`（相机画面首帧）
  - `time_scope`（可选，格式为 `start_time-end_time`）

### 3.2 预处理任务创建（按相机 ID）
- 输入：一个或多个相机 ID（支持并行传入多个 ID）,上面的输出中的id和time_scope字段
- 处理：
  - 根据 ID 定位对应视频文件
  - 执行预处理模拟逻辑：
    - 模拟截帧与发送远端处理流程
    - 固定 `sleep 100s` 作为任务耗时模拟
- 输出：
  - 返回 `task_id`
  - 返回任务初始状态（如 `pending`）与初始进度（`0`）

### 3.3 预处理状态查询
- 输入：`task_id`
- 处理：
  - 查询任务当前状态（如 `pending`、`running`、`completed`、`failed`）
  - 返回任务进度（0~100）
  - 任务完成后返回按相机 ID 聚合的结果
- 输出：
  - `task_id`
  - `status`
  - `progress`
  - `camera_ids`
  - `results`（可选，任务未完成时可为空）

### 3.4 特征检索（文字 / 图片 / 图片+文字）
- 输入：
  - `mode`：`text` / `image` / `image_text`
  - `query`（可选）：兼容文本检索输入
  - `text`（可选）：用户文字特征描述
  - `image`（可选）：用户上传图片或图片 URL
- 处理：
  - 根据 `mode` 执行对应检索逻辑
  - 在已加载的 Mock 相机图片数据中返回相关结果
  - 结果按相关性排序（Mock 可采用规则或随机打分）
- 输出：检索结果列表，每项至少包含：
  - `camera_id`
  - `location`（相机位置）
  - `name`（相机名称）
  - `image_url`
  - `time_stamp`（时间戳）
  - `score`（可选，Mock 排序分）

## 4. 接口约束

### 4.1 相机搜索接口
- 路径：`/api/v1/cameras/search`
- 请求参数：
  - `query`：自然语言检索文本
- 返回结果：
  - `cameras`：相机列表（`id`、`location`、`image_url`）

### 4.2 预处理任务创建接口
- 路径：`/api/v1/cameras/preprocess`
- 请求参数：
  - `camera_ids`：相机 ID 数组
- 返回结果：
  - `task_id`
  - `status`
  - `progress`
  - `camera_ids`

### 4.3 预处理状态查询接口
- 路径：`/api/v1/cameras/preprocess/{task_id}`
- 请求参数：
  - `task_id`（路径参数）
- 返回结果：
  - `task_id`
  - `status`
  - `progress`
  - `camera_ids`
  - `results`（按 ID 聚合）

### 4.4 特征检索接口（统一）
- 路径：`/api/v1/search`
- 请求参数：
  - `mode`：`text` / `image` / `image_text`
  - `query`（可选）
  - `text`（可选）
  - `image`（可选，支持上传或 URL）
- 参数约束：
  - 当 `mode=text` 时，`text` 或 `query` 必填
  - 当 `mode=image` 时，`image` 必填
  - 当 `mode=image_text` 时，`text` 与 `image` 均必填
- 返回结果：
  - `items`：检索结果列表，每项包含：
    - `camera_id`
    - `location`
    - `image_url`
    - `score`（可选，Mock 排序分）

## 5. 异常与边界处理

- 相机配置目录不存在或为空：返回明确错误信息
- 相机检索无匹配结果：返回空数组，不作为错误
- 相机 ID 不存在：对应 ID 返回未找到状态，不影响其他 ID 正常处理
- 视频文件缺失或命名不匹配：返回可定位问题的错误信息
- 任务 ID 不存在：状态查询接口返回明确未找到错误
- 预处理失败：返回失败原因并标识对应相机 ID
- 特征检索参数不完整或模式不匹配：返回明确参数错误信息

## 6. 非功能要求

- 支持批量 ID 的预处理任务处理
- 任务进度可查询，状态可追踪
- 支持特征检索三种输入模式：文字、图片、图片+文字
- 接口响应结构稳定、字段命名统一
- 错误信息可读、可用于联调定位

## 7. 验收标准

- 能成功加载配置目录中的相机数据
- 输入自然语言查询后，能返回符合条件的相机列表，包含 `id`、`location`、`image_url`
- 输入多个相机 ID 后，能返回预处理任务 `task_id`，并在任务执行期间可查询到状态与进度
- 预处理模拟流程耗时包含 `sleep 100s`
- 特征检索接口支持 `text`、`image`、`image_text` 三种模式并返回 `camera_id`、`location`、`image_url`
- 对无效 ID、缺失视频、无效任务 ID 等场景返回预期错误信息
